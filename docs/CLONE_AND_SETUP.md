# Madhu-Siri Clone And Setup

The app is ready to install and use as APK, just download, install and use [App](/Madhu_Siri/app/release/app-release.apk)

## Goal

This guide is for anyone cloning the project and running it in Android Studio without guessing where config values go.

## Prerequisites

- Android Studio
- Android SDK installed
- Google Play Services on the emulator or device
- A Google Maps Android API key

## 1. Clone and open

1. Clone the repository.
2. Open `AndroidStudioProjects/Madhu_Siri` in Android Studio.
3. Wait for Gradle sync to finish.

Gradle should fetch project dependencies automatically.

## Optional fast review path

If the repository includes a built APK, a reviewer does not need Android Studio to try the app.

Expected APK location:

- `app/build/outputs/apk/release/app-release.apk`

Reviewer steps:

1. Download or clone the repository.
2. Copy the APK to an Android phone.
3. Install the APK.
4. Open the app and test it directly.

This path is intended for internship evaluation or demo review.

## 2. Add local Maps config

Open or create `local.properties` in the project root and add:

```properties
MAPS_API_KEY=your_google_maps_api_key
```

Keep `sdk.dir=...` in the same file as generated by Android Studio.

`local.properties` is ignored by git, so this stays local.

## 3. Firebase config

The project already includes:

- `app/google-services.json`

That file is enough for the checked-in Firebase project connection, but Google Sign-In still depends on the right SHA fingerprints being added in Firebase for the machine and build type you use.

## 4. Run debug build

1. Connect a device or start an emulator.
2. Run the `app` configuration.
3. Grant location and notification permissions when prompted.

## 5. Enable Google Sign-In locally

If Google Sign-In does not complete:

1. Open Firebase Console.
2. Open the Android app entry for `com.example.madhu_siri`.
3. Add your debug SHA-1 and SHA-256.
4. Re-download `google-services.json`.
5. Replace `app/google-services.json`.
6. Rebuild the app.

## 6. Build a release APK or AAB

Release packaging also needs local signing files:

- `keystore.properties`
- `app/keys/madhu-siri-release.jks`

These are intentionally not committed.

## 7. What is safe to push to GitHub

Safe:

- source code
- docs
- Gradle files
- checked-in Firebase config, if this is still your intended shared demo project
- the built review APK, if you want reviewers to install the app directly

Do not push:

- `local.properties`
- `keystore.properties`
- any `.jks` signing file
- any unrestricted billing-enabled Maps key

## 8. Important reality

The project can be cloned, synced, and understood immediately from the repo, but a fully working map and release build still require local config values by design. That is the correct tradeoff if you do not want other people using your billing key or release signing identity.
